Speech Command and Speaker Recognition with CNNs and GMMs

Ben Bellenchia*, Kai Perez*, Brian Patterson*, Eric Dutan*
UAlbany CNSE ECE465 - Machine Learning Project
Showcase Day 2026
*Indicates Equal Contribution

Audio clips are converted into mel spectrogram images so a CNN can classify spoken commands. When the command “go” is detected, MFCC features are passed to a GMM speaker model to identify which team member spoke the activation word.

Abstract

This project presents a MATLAB-based speech command and speaker recognition system. One-second, 16 kHz audio clips are converted into normalized 64 × 64 mel spectrogram images and classified with a convolutional neural network (CNN). The final CNN model, modelv5, recognizes the spoken commands “go,” “no,” “stop,” and “yes,” while also detecting silence and using an “unknown” class to reject uncertain inputs. The command dataset was split into 70% training, 15% validation, and 15% testing data, with random time shifting, gain variation, and background-noise augmentation added to improve robustness.

The updated demo adds a second stage for speaker identification. After the CNN detects the activation command “go”, the same recording is processed using Mel-frequency cepstral coefficient (MFCC) features and compared against one Gaussian Mixture Model (GMM) per team member. The GMM stage returns the speaker with the highest log-likelihood score, while a score-gap threshold can reject uncertain speaker matches. This extension turns the project from command recognition alone into a two-stage system: first deciding what command was spoken, then identifying who spoke it. The speaker model works best when training and demo recordings are collected with a consistent microphone setup, highlighting an important real-world limitation for robust voice recognition in noisy or device-variable environments.

System Overview

The updated pipeline uses two separate models. The CNN performs general command recognition from mel spectrogram images, while the GMM stage is only activated after the command go is detected. This keeps the speaker model focused on the recordings it was trained for, since the speaker dataset was collected from team members saying the same activation word.

  1. Record audio: MATLAB records a one-second, 16 kHz audio sample.
  2. Predict command: The audio is converted into a 64 × 64 mel spectrogram and classified by the CNN.
  3. Check confidence: Predictions below the confidence threshold are labeled as unknown.
  4. Identify speaker: If the command is go, MFCC features are scored with a GMM speaker model.

Model Results

The final CNN model, modelv5, classifies go, no, stop, and yes, while also handling silence and uncertain inputs with an unknown class. Audio clips are one second long at 16 kHz, normalized, converted into 64 × 64 mel spectrograms, and split into 70% training, 15% validation, and 15% testing data.

The final architecture uses three convolution layers with batch normalization, max pooling, dropout, and fully connected classification layers. Earlier four-command models reached about 93.9% accuracy on the cleaner command-only task, while the final model reached about 91.6% validation accuracy after adding noise augmentation, silence detection, and unknown-input rejection for a more realistic speech-command system.

The newest update adds a GMM speaker-recognition stage for the command go. MFCC features are extracted from the live recording and scored against one GMM per speaker. This allows the demo to report both the recognized command and the predicted speaker.

Final CNN training progress: validation accuracy reached about 91.6% after adding noise augmentation, silence handling, and an unknown class.

Project Demonstration

The MATLAB demo records a one-second audio sample, normalizes the signal, converts it into a 64 × 64 mel spectrogram, and passes it through the trained CNN to predict the spoken command. If the command is detected as go, the demo then runs a GMM speaker-recognition stage to identify who spoke the activation word.

The command classifier reports the predicted label and confidence score. If the CNN confidence is below 75%, the result is labeled as unknown rather than forcing the recording into one of the command classes. When go is accepted, MFCC features are extracted from the same audio and scored against the trained speaker GMMs. The speaker with the highest log-likelihood score is selected, unless the gap between the best and second-best score is too small.

Live demo output: the CNN identifies the command as “go,” then the MFCC-GMM speaker model identifies the speaker as Ben.

Model Progression

Initial CNN baseline: about 83.9% validation accuracy.

Intermediate model: added deeper CNN layers and improved validation accuracy.

Final model: added noise augmentation, dropout, silence, and unknown handling for a more realistic speech-command system.

Limitations and Future Work

The CNN command model includes noise augmentation and unknown-input handling, but the GMM speaker model is more sensitive to recording conditions. Testing showed that microphone differences can significantly affect speaker scores, because MFCC features capture both voice characteristics and recording-device effects.

Future improvements would include collecting speaker recordings from multiple microphones, adding more background-noise augmentation, testing in busier acoustic environments, and comparing the GMM approach against more advanced speaker-embedding methods.
